GitHub Manager Guide

From Zero to Hero: A complete manual for everyone.

👋 Welcome!

Hello friend! If you are reading this, you are about to start a journey into the world of GitHub using the GitHub Manager tool. Whether you are a sighted user or using a screen reader like NVDA or JAWS, this guide is written specifically for you.

We know GitHub can look scary and complicated. It talks about "Commits," "Repos," and "Branches." Don't worry. This program was built to hide the scary parts and give you a simple, clean window to manage your files and projects.

🤔 What is GitHub anyway?

Imagine you are writing a book or coding a program. You save a file today. Tomorrow, you change it. The next day, you delete a paragraph. Suddenly, you realize the paragraph you deleted yesterday was actually important!

GitHub is like a magical time machine for your files. It's a cloud storage (like Google Drive or Dropbox), but with superpowers:

  • It saves every version of your file (History).
  • It lets you share your work with the world (Open Source).
  • It lets people report problems with your work (Issues).
  • It lets you package your work for download (Releases).

In GitHub, a project folder is called a Repository (or "Repo" for short).

🔑 Step 1: The Key to the Castle (Getting a Token)

Before the program can talk to GitHub, it needs permission. GitHub doesn't use your normal password for apps anymore; it uses something called a Personal Access Token (PAT). Think of this as a special, long password generated just for this app.

Important: You only need to do this once. Once the program saves the token, you are good to go!

How to get your Token (Step-by-Step):

  1. Go to GitHub Tokens Settings (You will need to sign in).
  2. Look for a button that says "Generate new token" (Choose "Classic" if asked).
  3. Note name: Type "GitHub Manager App" (or anything you like).
  4. Expiration: Choose "No expiration" if you don't want to do this again next month.
  5. Select scopes (Permissions): This is crucial. You will see a list of checkboxes. You must check:
    • repo (Full control of private repositories)
    • user (Update all user data)
    • delete_repo (If you want to be able to delete projects)
  6. Scroll to the bottom and click "Generate token".
  7. COPY IT NOW! You will see a long string of random characters starting with ghp_. Copy it immediately. GitHub will never show it to you again.

🦮 For Screen Reader Users (NVDA/JAWS):

GitHub's interface can be busy. Here is a quick tip:

  • Navigate by Buttons (Press B) to find "Generate new token".
  • Use Checkboxes (Press X) to check the permissions mentioned above.
  • When the token is generated, it usually appears near the top of the content area. Navigate by lines or headings to find the string starting with "ghp_".

Entering the Token:

Open GitHub Manager. If it's your first time, it will ask for the token. Paste it in the box and click Save (or press Enter). The file gh_token.txt will be created to remember you.

🖥️ The Interface & Features

The program is divided into two main areas: The Dashboard (Main Window) and the Repository Tab (Inside a project).

1. The Dashboard (Main Window)

This is your home base. Here you see a list of all your projects (Repositories).

  • My Repos: Refreshes the list to show your projects.
  • Advanced Search: Lets you find other people's code or users on GitHub.
  • Create Repo: Starts a brand new blank project on GitHub.

2. The Repository Tab (Inside a Project)

When you press Enter on a repo in the list, a new tab opens. This is where the magic happens.

📂 Files & Folders

You will see a list of files. You can navigate them just like Windows Explorer.

  • Open File: Pressing Enter on a file downloads it to a temporary folder and opens it on your computer.
  • Edit Internal: (Ctrl+Enter) Opens a simple text editor inside the program to make quick changes and save them back to GitHub immediately.

🌿 Branches

You will see a dropdown menu for "Branches". Imagine a Branch as a "Parallel Universe". The main branch is the real world. You can create a new branch to test crazy ideas without breaking the main code.

🐛 Issues (Problems & To-Do)

What it is: On GitHub, "Issues" are like a forum or a To-Do list for that specific project.
In the App: You can read discussions, reply to them, close solved problems, or create new ones.

📦 Releases (The Finished Product)

What it is: When a programmer finishes a version (like v1.0), they "Release" it.
In the App: You can see all versions, read what's new (Release Notes), and download the attached files (Assets) like ZIPs or EXEs.

📜 History (Time Machine)

Every time you save a file on GitHub, it creates a "Commit". The History window shows you who changed what and when.

❓ FAQ & Troubleshooting

The app says "Repo Not Found" when I try to open a link.

Make sure the link is correct. It should look like owner/repo-name or a full URL. Also, ensure your Token has permission to see private repos if the link is private.

I see "Binary File Error".

This happens if you try to "Edit" an image, a PDF, or an EXE file inside the app. The internal editor only works for text code. Use Enter or Ctrl+D to download and view these files instead.

Does this update automatically?

Yes! Every time you open the app, it checks a secure file (data.dat) connected to the server. If there is a new version, it will ask you if you want to update.
